Events∵

You can configure events to be published via a POST to the endpoint of your choice via callbacks. Once configured, events are accessible via the events endpoint.

note
Balanced may deliver more than one event with the same payload, but with a different event id.

Event types∵

All transactional resources (accounts, holds, credits etc) are evented. The format of the type field is resource.event_type where event_type is one of created, updated, deleted, as well as some transaction specific event types succeeded, failed, and canceled.

Transactional Events∵

credit.canceled
Occurs whenever a credit is canceled.
credit.created
Occurs whenever a new credit is created.
credit.failed
Occurs whenever a credit is rejected.
credit.succeeded
Occurs whenever a credit is successfully captured or deposited.
credit.updated
Occurs whenever a credit is updated.
debit.canceled
Occurs whenever a debit is canceled.
debit.created
Occurs whenever a new debit is created.
debit.failed
Occurs whenever a debit is rejected.
debit.succeeded
Occurs whenever a debit is successfully captured or withdrawn.
debit.updated
Occurs whenever a debit is updated.
hold.captured
Occurs whenever a hold is captured.
hold.created
Occurs whenever a new hold is created.
hold.updated
Occurs whenever a hold is updated.
hold.voided
Occurs whenever a hold is voided.
refund.canceled
Occurs whenever a refund is canceled.
refund.created
Occurs whenever a new refund is created.
refund.failed
Occurs whenever a refund is rejected.
refund.succeeded
Occurs whenever a refund is successfully captured or deposited.
refund.updated
Occurs whenever a refund is updated.
reversal.created
Occurs whenever a reversal is created.
reversal.failed
Occurs whenever a reversal is rejected.
reversal.succeeded
Occurs whenever a reversal is successfully completed.

Funding Instrument Events∵

bank_account.created
Occurs whenever a new bank account is created.
bank_account.invalidated
Occurs whenever a bank account is invalidated.
bank_account.updated
Occurs whenever a bank account is updated.
bank_account_verification.created
Occurs whenever a bank account verification has been created.
bank_account_verification.deposited
Occurs whenever bank account verification micro-deposits have been successfully sent.
bank_account_verification.failed
Occurs whenever bank account verification has failed.
bank_account_verification.succeeded
Occurs whenever bank account has been successfully verified.
bank_account_verification.updated
Occurs whenever a bank account verification has been updated.
bank_account_verification.verified
Occurs whenever bank account has been successfully verified.
card.created
Occurs whenever a new card is created.
card.invalidated
Occurs whenever a card is invalidated.
card.updated
Occurs whenever a card is updated.

Retrieve an Event∵

Retrieves the details of an event that was previously created. Use the uri that was previously returned, and the corresponding event information will be returned.

Definition

GET https://api.balancedpayments.com/events/:event_id

balanced.Event.find

Balanced::Event.find()

Balanced\Event::get

Event(String uri)

Example Request

curl https://api.balancedpayments.com/v1/events/EV547b9dfcb91a11e3856302dbb40573f0 \
     -u ak-test-2755A6Q56FwdnGMNXK72fHjCSXXlruJe6:

import balanced

balanced.configure('ak-test-14W5azoiV99O1XiPwZ3faH10MaUdZ1kCA')

event = balanced.Event.find('/v1/events/EV02f1fad84d4711e384a9026ba7d31e6f')

require 'balanced'
Balanced.configure('8c3aeeb80e9e11e38901026ba7f8ec28')

event = Balanced::Event.find('/v1/events/EV8c8e419e0e9e11e39281026ba7d31e6f')

require(__DIR__ . '/vendor/autoload.php');

$event = Balanced\Event::get("/v1/events/EV9be5771e4d4811e38afd026ba7d31e6f");

?>

List all Events∵

limit
optional integer. Defaults to 10.
offset
optional integer. Defaults to 0.

Definition

GET https://api.balancedpayments.com/events

balanced.Event.query

Balanced::Event.all

\Balanced\Event->all()

Event.Collection(String uri)

Example Request

curl https://api.balancedpayments.com/v1/marketplaces/TEST-MP2zmzeddBrHjK5zoThHGpzi/events?limit=2 \
    -u ak-test-2755A6Q56FwdnGMNXK72fHjCSXXlruJe6:

import balanced

balanced.configure('ak-test-14W5azoiV99O1XiPwZ3faH10MaUdZ1kCA')

events = balanced.Event.query.all();

require 'balanced'
Balanced.configure('8c3aeeb80e9e11e38901026ba7f8ec28')

marketplace = Balanced::Marketplace.my_marketplace
events = Balanced::Event.all(:limit => 2)

<?php

require(__DIR__ . '/vendor/autoload.php');

Httpful\Bootstrap::init();
RESTful\Bootstrap::init();
Balanced\Bootstrap::init();

Balanced\Settings::$api_key = "ak-test-2KZfoLyijij3Y6OyhDAvFRF9tXzelBLpD";

$marketplace = Balanced\Marketplace::mine();
$events = $marketplace->events->query()->all();

?>

Balanced.configure("ak-test-1p1Tsac7gHeMQowL2seB7ieliuAJAufyq");

Event.Collection events = new Event.Collection("/v1/events");
